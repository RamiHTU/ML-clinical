ML term: task definition

The clinical problem

Setup

Can we predict a patient's systolic blood pressure from their age alone?

No mathematics yet — just the clinical motivation. We start with a question a clinician would actually ask.

What we know clinically

Doctors already observe that older patients tend to have higher blood pressure. This is established clinical knowledge.

What we want to do

Can a machine learn that same pattern automatically — just from looking at past patient data?

This is the starting point of every ML project: a real problem, not an algorithm. The algorithm comes much later.
Slide 2
ML term: supervised learning ML term: hypothesis function

Problem statement

Setup

Plain English

Given a patient's age, predict their systolic blood pressure.

Mathematical

BP = f(age)

Goal: find f  such that  f(age) ≈ BP

What does f mean?

f is a function — a rule or formula that takes age as input and returns a predicted blood pressure. The job of machine learning is to find the best f from data.

Because we already know the correct BP for each training patient, this is called supervised learning — the model learns under supervision of known answers.
Slide 3
ML term: train set ML term: test set Key concept: train/test split

Before we look at data — a crucial idea

Setup
We must decide how to split our data before training begins. This is one of the most important rules in machine learning.

Training set

The rows the model learns from. It sees these during training and adjusts its parameters based on them.

Test set

Rows held back entirely. The model never sees these during training. Used only for final evaluation.

The medical student analogy

A medical student studies past cases (train set), then sits an exam on new cases they have never seen before (test set). You would not mark them on cases they already memorised — that tells you nothing about whether they truly learned.

Rule: the model must never see test data during training — ever. Breaking this rule gives you falsely optimistic results.

Slide 5
ML term: model ML term: parameter

The model

Training
BP = w × age

What is w?

w is the parameter — the single number the model needs to learn. Different values of w give different lines through the data. The model's job is to find the best w.

Why a line?

Look at the scatter plot — the relationship appears roughly linear. As age increases, BP increases. A line is the simplest model that captures this.

Note on the bias term: a complete model would be BP = w × age + b, where b is a baseline constant. We are deferring b for now to keep things clear — we will add it in the next lesson.

Slide 8
ML term: optimisation ML term: learning

Finding the best w

Training

The model "learns" by searching for the w that produces the lowest average error.

In practice this search is done automatically — the algorithm does not try every value by hand. It follows the slope of the cost curve downward until it reaches the minimum.

Cost curve — average error vs w (the model finds the bottom)

The automatic process of finding the minimum is called optimisation. The most common method is called gradient descent — but you do not need to know the mechanics yet. The key idea: the model searches for the w at the bottom of this curve.

Evaluation — the honest test

1.0 × 45 = 45 mmHg

Actual: 127 mmHg

Error: |45 − 127| = 82 mmHg

High error — because we have no bias term b yet. This is expected at this stage.

What evaluation tells us

Did the model truly learn the pattern, or did it just memorise the 3 training rows? Testing on a new patient it has never seen gives an honest answer.

Overfitting (preview)

If we evaluated only on training data, the model could score perfectly by memorising — without learning anything useful. This is called overfitting. The test set catches it.
