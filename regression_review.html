The clinical question

What is simple linear regression?

Can we predict a patient's systolic blood pressure from their age alone?

This is the question that motivates simple linear regression. We have one input (age) and one output (blood pressure), and we want to find a mathematical rule that maps one to the other.

Why "regression"?

Regression is the family of ML models that predict a continuous number — like blood pressure, blood glucose, or cholesterol level. If the output were a category (disease / no disease), we would use classification instead.

Why "simple"?

Simple linear regression uses exactly one input feature to make predictions. When we use two or more features — like age and BMI together — it becomes multiple regression.

The big idea: we assume there is a roughly linear relationship between age and blood pressure — that is, as age increases, BP increases at a roughly constant rate. The model learns that rate from past patient data.

Feature vs label — a reminder

The feature (also called the input or x) is what we give the model — in this case, a patient's age.
The label (also called the target or y) is what we want to predict — in this case, systolic blood pressure.

The model equation

What does the model look like?
Click on any term in the formula below to see what it means.
ŷ  =  w  ×  age  +  b
w
Coefficient (slope)
How much BP changes for every 1-year increase in age. This is what the model learns from the data. Also called the weight or slope.
b
Intercept (baseline)
The predicted BP when age = 0. Not clinically meaningful on its own, but needed mathematically to position the line correctly.
age
Feature (input)
The value we give the model to make a prediction. In this case, a patient's age in years. Also written as x.
ŷ
Predicted output
The model's prediction of systolic BP. The hat (^) means it is an estimate, not the true measured value. Read as "y-hat".
Example: if the model learns w = 0.85 and b = 70, then for a patient aged 50:
ŷ = 0.85 × 50 + 70 = 42.5 + 70 = 112.5 mmHg

Effect of w

A larger w = steeper slope. The line rises faster as age increases. A smaller w = flatter line.

Effect of b

A larger b shifts the entire line up. A smaller b shifts it down. The slope stays the same.

Cost curve — error vs w

The cost function is the formula for the average error. In this example we are using Mean Absolute Error (MAE): the average of |predicted − actual| across all patients. The model's goal is to find the w that puts this number as low as possible.

What the model learns

Finding the best parameters automatically

Training is the process of finding the parameters (w and b) that produce the lowest possible cost.

The model does not try every possible value of w by hand. Instead, it uses an algorithm called gradient descent — which follows the slope of the cost curve downward, step by step, until it reaches the bottom.

Cost curve — the model finds the minimum

w = 0.3
Cost ≈ 37.8 mmHg

Line too flat — misses older patients badly

w = 0.85
Cost ≈ 6.2 mmHg

Best fit — line passes close to all points

Gradient descent in one sentence: start with a random w, check which direction the cost decreases, take a small step in that direction, repeat — until you reach the bottom of the cost curve. In scikit-learn, this happens automatically when you call model.fit().

Evaluation metrics

How do we know if the model is any good?
We always evaluate on the test set — patients the model has never seen. These three metrics each answer a slightly different question.
MAE
Mean Absolute Error
avg( |ŷ − y| )
The average size of prediction errors, in the same units as the target. Easy to interpret — if MAE = 8, predictions are off by 8 mmHg on average.
Errors: 5, 9, 8
MAE = (5+9+8)/3 = 7.3 mmHg
RMSE
Root Mean Squared Error
√( avg( (ŷ−y)² ) )
Similar to MAE but penalises large errors more. Because errors are squared, a single large miss hurts the score more than many small ones.
Errors: 5, 9, 8
RMSE = √((25+81+64)/3) = 7.7 mmHg
R²
R-squared (R²)
1 − (SS_res / SS_tot)
Proportion of BP variation the model explains. 0 = explains nothing, 1 = perfect. R² = 0.85 means the model explains 85% of variation in blood pressure.
R² = 0.85
→ 85% of BP variation explained by age

Which metric should you use?

Use MAE when you want a simple, interpretable error in clinical units. Use RMSE when large errors are especially dangerous — for example, missing a high-risk patient by a lot is worse than missing many patients by a little. Use R² when you want to know how much of the outcome your features explain overall.

Important: R² = 0.85 sounds good — but always ask good compared to what? A model predicting age → BP might score 0.85. But if a clinician can eyeball a scatter plot and do just as well, the model adds little value. Context always matters.
