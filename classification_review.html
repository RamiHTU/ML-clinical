From Regression to Classification

Regression

  • Predicts a continuous number
  • How many days will this patient stay?
  • What will this patient's WBC be?
  • Output: 7.3 days, 11.2 × 10³/µL

Classification ← Today

  • Predicts a category or label
  • Will this patient be readmitted?
  • Is this tumor malignant or benign?
  • Output: Yes / No / Stage I–IV
Clinical connection: Most clinical decisions are classification decisions — diagnosis, triage, risk stratification. ML gives us a systematic way to automate and scale these judgements.

Q1 A model is trained to predict whether a cancer patient will develop febrile neutropenia within 30 days of chemotherapy. This is best described as a:

✓ Correct. The outcome has exactly two categories (develops febrile neutropenia: Yes / No), making this a binary classification problem. The fact that the inputs are lab values doesn't determine the problem type — the output does.

Q2 A model assigns each patient to one of four triage levels: Low, Medium, High, or Critical. This is an example of:

✓ Correct. Four possible category labels (Low / Medium / High / Critical) makes this multi-class classification. Binary classification would only have two options.
✗ Not quite. When the output is a category — and there are more than two categories — it's multi-class classification. Regression produces a number; binary classification has two outcomes; unsupervised learning has no labels at all.

Decision Boundaries: Linear vs. Non-Linear

What is a Decision Boundary?

A decision boundary is the line, curve, or surface a classifier draws in feature space to separate one class from another. Everything on one side gets one label; everything on the other side gets a different label.

Linear Boundary

  • A straight line (2D) or flat plane (higher dims)
  • Clinical analogy: "Refer if PSA > 4 ng/mL"
  • Simple, interpretable, may underfit
  • Used by: Logistic Regression

Non-Linear Boundary

  • A curve or complex shape
  • Captures interactions between features
  • More flexible, risk of overfitting
  • Used by: Decision Trees, Neural Networks, SVMs
Key insight: A linear boundary is like a single clinical cutoff. A non-linear boundary is like clinical gestalt — weighing many factors in complex, interacting ways. More complex is not always better.

Q3 A clinician says: "We flag a patient as high-risk if their ECOG score is above 2 AND their WBC is above 15." Which type of decision boundary does this most resemble?

✓ Correct. Using an AND condition between two features creates an interaction — the boundary in feature space is not a single straight line separating the classes. This kind of rule maps to a non-linear (specifically, axis-aligned rectangular) decision boundary.
✗ Not quite. Although each individual threshold (ECOG > 2, WBC > 15) is a linear rule, combining them with AND creates an interaction between the two features. The resulting boundary in 2D feature space is not a single straight line — it's an L-shaped region, which is non-linear.

Q4 Which of the following is the main risk of using a very complex (highly non-linear) decision boundary?

✓ Correct. Overfitting is the primary risk of complex non-linear models. The model learns the quirks and noise of the training set rather than true underlying patterns, so performance drops when applied to new patients.
✗ Not quite. A very complex boundary is more likely to overfit — memorising the training data, including its noise — rather than underfit. Underfitting is the risk of overly simple models, not complex ones.

The Threshold is a Clinical Decision

Most classifiers output a probability, not a label directly. You choose the threshold that matches your clinical priorities. Lowering the threshold (e.g. from 0.5 to 0.3) catches more at-risk patients but raises more false alarms. This tradeoff is a clinical — not just a statistical — choice.

Q5 A readmission classifier outputs a probability of 0.42 for a patient. The current classification threshold is 0.5. What label does the patient receive?

✓ Correct. The rule is simple: if P ≥ threshold → Positive; if P < threshold → Negative. Since 0.42 < 0.5, the patient is labelled Negative (Not Readmitted). The threshold is a hard cutoff — proximity doesn't matter.
✗ Not quite. The threshold acts as a hard cutoff. Since 0.42 is strictly below 0.5, the label is Negative (Not Readmitted). The model always produces a label — it doesn't abstain when values are close to the threshold.

Q6 The oncology team decides to lower the classification threshold from 0.5 to 0.3 to catch more at-risk patients. What is the most likely consequence of this change?

✓ Correct. Lowering the threshold means more patients cross it — so more are labelled Positive. This catches more true positives (higher sensitivity) but also captures more false positives. AUC is unaffected by the threshold; it summarises performance across all thresholds.
✗ Not quite. Lowering the threshold makes the classifier more "trigger-happy" — it flags more patients. This increases sensitivity (catches more true cases) but also increases false positives. Overall accuracy may actually decrease, and AUC doesn't change with threshold adjustment.

Sensitivity = TP / (TP + FN) — of all patients who WERE readmitted, how many did we catch?
Specificity = TN / (TN + FP) — of all patients who were NOT readmitted, how many did we correctly clear?

These are the same sensitivity and specificity you know from diagnostic testing — just applied to a model.

Q7 A readmission classifier is tested on 100 patients. 50 are truly at risk (Positive). The model flags 40 of them correctly, misses 10, raises 5 false alarms, and correctly clears 45 non-at-risk patients. What is TP?

✓ Correct. TP = True Positives = patients who were actually at risk AND the model correctly flagged. That's 40. The 10 missed cases are FN. The 5 false alarms are FP. The 45 correctly cleared are TN.
✗ Not quite. TP = patients the model correctly said were at risk. Out of 50 truly at-risk patients, the model caught 40 → TP = 40. The 10 it missed are False Negatives (FN = 10). The 5 false alarms are False Positives (FP = 5). The 45 correctly cleared are True Negatives (TN = 45).

Q8 In the context of a cancer screening model, which type of error is generally considered more clinically dangerous?

✓ Correct. A False Negative in cancer screening means a patient with cancer is told they are clear — they receive no follow-up or treatment. This missed diagnosis can be life-threatening. False Positives cause unnecessary anxiety and follow-up tests, which is a cost, but typically less dangerous.
✗ Not quite. In cancer screening, the most dangerous error is the False Negative — a cancer that is present but missed by the model. The patient gets no treatment. False Positives (unnecessary further testing) are a burden, but the patient ultimately gets follow-up care.

F1 Score
2 × P×R / (P+R)
Harmonic mean of precision and recall. Useful when both matter.

The Accuracy Trap

If 90% of patients are NOT readmitted, a model that always says "No" achieves 90% accuracy — but catches zero at-risk patients. Always report sensitivity alongside accuracy in clinical settings.

Q9 Using the KHCC numbers from the lecture: TP=38, FN=12, FP=22, TN=428. What is the sensitivity of the model?

✓ Correct. Sensitivity = TP / (TP + FN) = 38 / (38 + 12) = 38/50 = 0.76 = 76%. This tells us the model caught 76% of the patients who were truly at risk.
✗ Not quite. Sensitivity = TP / (TP + FN). We need the denominator to be all patients who were truly positive (at risk). That's TP + FN = 38 + 12 = 50. So sensitivity = 38/50 = 76%. Option B is precision. Option C is specificity. Option D is accuracy.

Q10 An oncologist says: "I need a model that never misses a true case, even if it raises some false alarms." Which metric should be maximised?

✓ Correct. "Never miss a true case" = maximise sensitivity (recall = TP / (TP + FN)). High sensitivity means few false negatives. The tradeoff is more false positives — but the oncologist has explicitly said they accept that.
✗ Not quite. "Never miss a true case" means minimising false negatives — which means maximising sensitivity (recall). Specificity is about correctly clearing negative cases. Precision is about the quality of positive predictions. Accuracy is a blunt overall measure.

ROC Curve & AUC

The ROC Curve

The ROC (Receiver Operating Characteristic) curve plots Sensitivity (TPR) on the Y-axis against 1 − Specificity (FPR) on the X-axis across all possible thresholds. A perfect model curves toward the top-left corner. The diagonal line represents random guessing.

AUC RangeInterpretationClinical Context
1.0 Perfect Never seen in real clinical data
0.9 – 1.0 Excellent Rare; very strong model
0.8 – 0.9 Good Many published oncology models
0.7 – 0.8 Acceptable Moderate; worth improving
0.5 – 0.7 Poor Little better than guessing
0.5 Random Coin flip — no clinical value
Clinical interpretation: An AUC of 0.88 means the model correctly ranks a randomly chosen readmitted patient above a non-readmitted patient 88% of the time. Changing the threshold does not change the AUC.

Q11 On a ROC curve, what does a point in the top-left corner (Sensitivity ≈ 1, FPR ≈ 0) represent?

✓ Correct. The top-left corner (Sensitivity = 1, FPR = 0) means the model catches every true case (no false negatives) while producing zero false positives. This is the ideal point — a perfect classifier sits here.
✗ Not quite. The top-left corner represents the best possible performance: Sensitivity = 1 (all positives caught) and FPR = 0 (no false alarms). That's a near-perfect classifier. The diagonal line represents random guessing. The bottom-right corner would represent always predicting positive.

Q12 A team trains two readmission models: Model A has AUC = 0.85 at threshold = 0.5. They then lower the threshold to 0.3. What happens to the AUC?

✓ Correct. AUC summarises model performance across all possible thresholds simultaneously — it is completely independent of any single threshold choice. Changing the threshold changes sensitivity and specificity at that operating point, but not the AUC.
✗ Not quite. AUC = Area Under the ROC Curve, which is computed by sweeping across all thresholds. It is a threshold-independent measure of overall discriminating ability. Changing a single operating threshold changes the sensitivity/FPR tradeoff at that point, but doesn't touch the AUC.

Class Imbalance

What Is Class Imbalance?

Class imbalance occurs when one outcome is much more common than the other. This is extremely common in oncology: adverse events, readmissions, and rare diagnoses are by definition rare. If 92% of patients are not readmitted, a model that always says "No" achieves 92% accuracy — but has zero clinical value.

Why It Matters

  • Accuracy is inflated and misleading
  • Sensitivity collapses toward zero
  • Model learns to always predict the majority class
  • Clinically dangerous if minority class = the sick patients

What Can Be Done

  • Oversample the minority class (e.g. SMOTE)
  • Undersample the majority class
  • Use class weights during training
  • Report sensitivity, not just accuracy

Q13 A dataset has 950 patients with no adverse event and 50 patients with an adverse event. A model predicts "No adverse event" for every patient. What is its accuracy?

✓ Correct. The model correctly predicts 950 out of 1000 patients (all the true negatives), so accuracy = 950/1000 = 95%. But it catches zero adverse events — its sensitivity is 0%. This perfectly illustrates why accuracy alone is insufficient with imbalanced data.
✗ Not quite. Accuracy = correct predictions / total predictions. The model says "No" for everyone, and 950 out of 1000 patients truly have no adverse event, so it gets 950 right → 95% accuracy. But its sensitivity = 0/50 = 0%. High accuracy, zero clinical utility.

Q14 When you receive the results of a new ML model for clinical use, which of the following is the most complete evaluation approach?

✓ Correct. No single metric tells the whole story. Accuracy can mislead with imbalanced data; AUC doesn't tell you about real-world operating performance; sensitivity and specificity alone don't reveal overall accuracy. A complete evaluation uses all of them together — and always checks class distribution first.
✗ Not quite. Each metric reveals a different dimension of model performance. With imbalanced clinical data, accuracy alone is dangerous; AUC alone doesn't tell you operational performance; precision alone ignores missed cases. The best practice is to report multiple metrics and always check class distribution before interpreting anything.
