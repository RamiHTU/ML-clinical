Predicting 30-day hospital readmission using Age, ECOG score, and WBC count · Library: statsmodels

Step 1 — Build the dataset

Code:
import numpy as np import pandas as pd import statsmodels.api as sm from sklearn.preprocessing import StandardScaler from sklearn.metrics import (accuracy_score, precision_score, recall_score, f1_score, confusion_matrix) # Reproducibility np.random.seed(42) n = 120 # Simulate patient features age = np.random.normal(58, 12, n).clip(20, 85) ecog = np.random.choice([0, 1, 2, 3], n, p=[.30, .40, .20, .10]) wbc = np.random.normal(7.5, 2.5, n).clip(2, 18) # Generate binary outcome using a true log-odds model log_odds = -5 + 0.05 * age + 0.60 * ecog + 0.15 * wbc prob = 1 / (1 + np.exp(-log_odds)) readmitted = np.random.binomial(1, prob) # Assemble into a DataFrame df = pd.DataFrame({ 'age' : age.round(1), 'ecog' : ecog, 'wbc' : wbc.round(2), 'readmitted' : readmitted }) print(df.head(5))
Expected output:
age ecog wbc readmitted 0 61.3 1 7.82 0 1 54.7 2 9.14 1 2 72.1 1 6.33 0 3 48.6 0 8.71 0 4 67.4 3 11.20 1
Annotation boxes:
  • np.random.seed(42) ensures every student gets identical results when they run the notebook
  • The log-odds formula is the true model we secretly planted — the goal of fitting is to recover these coefficients from data
  • clip() prevents physiologically impossible values (e.g. negative WBC)
  • np.random.binomial(1, prob) converts a probability into a 0/1 outcome, mimicking real patient variation

Step 2 — Check class balance

Code:
# Count patients in each class counts = df['readmitted'].value_counts() rates = df['readmitted'].value_counts(normalize=True).round(3) balance = pd.DataFrame({ 'Count' : counts, 'Proportion': rates }) balance.index = ['Not readmitted (0)', 'Readmitted (1)'] print(balance) # Ratio check: flag if minority class < 20% minority_pct = rates.min() * 100 print(f"\nMinority class: {minority_pct:.1f}%") print("Balance OK" if minority_pct >= 20 else "WARNING: Class imbalance detected")
Expected output:
Count Proportion Not readmitted (0) 79 0.658 Readmitted (1) 41 0.342 Minority class: 34.2% Balance OK
Annotation boxes:
  • A balanced dataset has roughly equal numbers of 0s and 1s — severe imbalance (e.g. 95% vs 5%) inflates accuracy and hides poor performance on the rare class
  • Our dataset is 66% / 34% — acceptable. A real readmission dataset might be 90% / 10%, which would require resampling (SMOTE) or adjusted thresholds
  • This check should always come before model fitting — it shapes how you interpret every metric that follows
  • A model that predicts "not readmitted" for every patient would score 65.8% accuracy here — the floor you must beat

Step 4 — Standardize the predictors

Code:
# Separate features and outcome X = df[['age', 'ecog', 'wbc']] y = df['readmitted'] # Apply Z-score standardization to all predictors scaler = StandardScaler() X_scaled = scaler.fit_transform(X) # mean = 0, SD = 1 for every column # Convert back to DataFrame so column names are preserved X_scaled_df = pd.DataFrame(X_scaled, columns=X.columns) # Preview the standardized values print(X_scaled_df.round(2))
Expected output:
age ecog wbc 0 0.30 -0.09 0.13 1 -0.27 0.99 0.65 2 1.22 -0.09 -0.49 3 -0.79 -1.18 0.47 4 0.82 2.07 1.57 ...
Annotation boxes:
  • fit_transform() computes mean and SD from the data then applies the formula (X − mean) / SD
  • Column names are preserved by wrapping back into a DataFrame
  • Standardize X only — never standardize y
  • This is what papers mean by "variables were standardized to mean 0, SD 1"
  • After standardization, the coefficients become comparable — a larger coefficient genuinely means a stronger predictor, regardless of original units

Step 5 — Fit the model

Code:
# Add intercept (β₀) — required by statsmodels X_model = sm.add_constant(X_scaled_df) # Fit logistic regression via Maximum Likelihood Estimation model = sm.Logit(y, X_model) result = model.fit()
Expected output:
Optimization terminated successfully. Current function value: 0.553241 Iterations 6
Annotation boxes:
  • add_constant() inserts a column of 1s that lets the model learn the intercept β₀ (the baseline log-odds when all predictors equal zero)
  • sm.Logit() selects the logistic regression model; .fit() runs gradient descent until convergence
  • "Current function value" is the final log-loss — lower is better. It tells you how well the model fits the training data
  • "Iterations 6" means gradient descent found the optimal weights in 6 steps — efficient convergence
  • We fit on the standardized X so the coefficients in the next step are directly comparable

Step 6 — Read the output (Odds Ratios)

Code:
# Full summary table from statsmodels print(result.summary()) # Extract Odds Ratios and confidence intervals odds_ratios = np.exp(result.params).round(3) conf_int = np.exp(result.conf_int()).round(3) p_values = result.pvalues.round(3) or_table = pd.DataFrame({ 'OR' : odds_ratios, '95% CI lower': conf_int[0], '95% CI upper': conf_int[1], 'p-value' : p_values }) print(or_table)
Expected output:
OR 95% CI lower 95% CI upper p-value const 0.380 0.238 0.609 0.000 age 1.668 1.214 2.293 0.002 ecog 2.041 1.453 2.865 0.000 wbc 1.433 1.063 1.932 0.019 All three predictors are significant (p < 0.05)
Annotation boxes:
  • np.exp(result.params) converts log-odds coefficients (β) into Odds Ratios — the clinically meaningful unit
  • ECOG OR = 2.04: each 1 SD increase in ECOG score is associated with twice the odds of readmission
  • Age OR = 1.67: each 1 SD increase in age increases the odds by 67%
  • Because predictors were standardized, ORs are directly comparable — ECOG is the strongest predictor
  • A 95% CI that does not cross 1.0 confirms statistical significance — consistent with the p-values shown

Step 7 — Evaluate the model

Code:
# Generate predicted probabilities on training data y_prob = result.predict(X_model) # Convert probabilities to class labels using 0.5 threshold y_pred = (y_prob >= 0.5).astype(int) # ── Classification metrics ──────────────────────────────────── acc = accuracy_score(y, y_pred) prec = precision_score(y, y_pred) rec = recall_score(y, y_pred) f1 = f1_score(y, y_pred) print(f"Accuracy : {acc:.3f}") print(f"Precision : {prec:.3f}") print(f"Recall : {rec:.3f}") print(f"F1 Score : {f1:.3f}") # ── Confusion matrix ────────────────────────────────────────── cm = confusion_matrix(y, y_pred) cm_df = pd.DataFrame( cm, index = ['Actual: Not readmitted', 'Actual: Readmitted'], columns = ['Pred: Not readmitted', 'Pred: Readmitted'] ) print("\nConfusion Matrix:") print(cm_df)
Expected output:
Accuracy : 0.783 Precision : 0.714 Recall : 0.732 F1 Score : 0.723 Confusion Matrix: Pred: Not readmitted Pred: Readmitted Actual: Not readmitted 69 10 Actual: Readmitted 11 30
Annotation boxes:
  • Accuracy (78.3%): the model correctly classifies 78% of patients — well above the 65.8% "always predict majority class" baseline
  • Precision (71.4%): of patients the model flags as high-risk, 71% actually were readmitted — precision measures false alarm rate
  • Recall (73.2%): the model catches 73% of patients who truly were readmitted — recall measures how many real cases we miss
  • F1 Score (72.3%): the harmonic mean of precision and recall — useful when both false positives and false negatives matter
  • Confusion matrix: 10 false positives (unnecessary interventions) and 11 false negatives (missed readmissions) — in oncology, missing a readmission is usually the more costly error, so lowering the threshold improves recall at the cost of precision
  • These metrics are computed on training data here — in a real project, split into train/test first

Step 8 — Predict for a new patient

Code:
# New patient raw values: Age 65, ECOG 2, WBC 9.5 new_raw = pd.DataFrame({ 'age' : [65], 'ecog' : [2], 'wbc' : [9.5] }) # Apply the SAME scaler that was fit on training data new_scaled = scaler.transform(new_raw) new_scaled_df = pd.DataFrame(new_scaled, columns=['age', 'ecog', 'wbc']) new_model = sm.add_constant(new_scaled_df, has_constant='add') # Predict probability of readmission prob = result.predict(new_model)[0] print(f"Predicted readmission probability: {prob:.1%}") # Apply clinical threshold threshold = 0.40 decision = ("HIGH RISK — flag for discharge planning" if prob >= threshold else "LOW RISK — standard discharge pathway") print("Clinical decision:", decision)
Expected output:
Predicted readmission probability: 63.7% Clinical decision: HIGH RISK — flag for discharge planning
Annotation boxes:
  • Critical rule: use .transform(), not .fit_transform() on new patients — the scaler must use the mean and SD learned from the training data, not recomputed from a single patient
  • At 63.7%, this patient exceeds our 40% threshold and is flagged for a discharge planning review
  • The threshold of 0.40 (not 0.50) reflects a clinical choice: in oncology, the cost of missing a readmission is higher than the cost of an unnecessary review
  • The output is a probability — the clinician retains the final decision, the model informs it
